Authenticating requests

To authenticate requests, include an Authorization header with the value "Bearer YOUR_AUTH_TOKEN_HERE".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

使用 Laravel Sanctum Bearer Token 進行認證。請在請求頭中包含：Authorization: Bearer {token}

取得認證用戶資訊的路由 需要 Sanctum 認證

Example request:
curl --request GET \
    --get "http://localhost/api/user" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json"

    "http://localhost/api/product-categories" \

    --header "Accept: application/json" \

    \"slug\": \"amniihfqcoynlazghdtqt\",
    \"parent_id\": 17,
    \"position\": 45,
    \"status\": true,
    \"description\": \"Amet iste laborum eius est dolor dolores.\",
    \"meta_title\": \"tdsufvyvddqamniihfqco\",
    \"meta_description\": \"ynlazghdtqtqxbajwbpil\"
}"

const url = new URL(
    "http://localhost/api/product-categories"
);

    "Content-Type": "application/json",
    "Accept": "application/json",
};

Request      

POST api/product-categories

Headers

Content-Type      

Example: application/json

Accept      

Example: application/json

Body Parameters

name   string   

Must not be greater than 100 characters. Example: vmqeopfuudtdsufvyvddq

slug   string  optional  

Must match the regex /^[a-z0-9]+(?:-[a-z0-9]+)*$/. Must not be greater than 100 characters. Example: amniihfqcoynlazghdtqt

parent_id   integer  optional  

The id of an existing record in the product_categories table. Example: 17

position   integer  optional  

Must be at least 0. Example: 45

status   boolean  optional  

Example: true

description   string  optional  

Must not be greater than 1000 characters. Example: Amet iste laborum eius est dolor dolores.

meta_title   string  optional  

Must not be greater than 100 characters. Example: tdsufvyvddqamniihfqco

meta_description   string  optional  

Must not be greater than 255 characters. Example: ynlazghdtqtqxbajwbpil
