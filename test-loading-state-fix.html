🔄 LomisX3 載入狀態修復測試

測試頁面刷新時的載入狀態和 UI 一致性

❌ 修復前的問題

問題描述：
• 頁面刷新後，會有短暫的時間顯示錯誤的 UI 狀態
• 使用者可能看到登入頁面或未認證狀態的閃爍
• AuthStore.initialize() 是異步的，但 UI 沒有等待其完成
• 導致認證狀態檢查時出現競態條件 (Race Condition)

✅ 修復方案

解決方案：
1. 在 App.tsx 中讀取 authStore 的 isLoading 狀態
2. 當 isLoading = true 時，顯示全頁載入畫面
3. 只有在 initialize() 完成後，才渲染主要應用內容
4. 防止 UI 狀態不一致和閃爍問題

🔄 載入流程演示

修復後的載入流程：

1. 用戶刷新頁面 → App.tsx 載入
2. useAuthStore() 讀取 isLoading 狀態
3. isLoading = true → 顯示載入畫面
4. useEffect 觸發 initialize() 執行
5. initialize() 檢查 localStorage 中的 token
6. 如有 token → 調用 /api/auth/me 驗證
7. 驗證成功 → 設定 user, isAuthenticated = true
8. initialize() 完成 → isLoading = false
9. 載入畫面消失 → 渲染主要應用內容
10. ProtectedRoute 檢查 isAuthenticated
11. 認證通過 → 顯示主頁面
